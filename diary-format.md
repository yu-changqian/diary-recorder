# 日记格式规范

## 每日日记模板

每日日记文件为 `$DIARY_HOME/daily/YYYY-MM-DD.md`，结构如下：

```markdown
---
date: YYYY-MM-DD
weekday: 星期X
entry_count: N
tags: [tag1, tag2, tag3]
---

# YYYY-MM-DD 星期X

## 〇 日志

### HH:MM #tag1 #tag2

用户原文内容，一字不改。

### HH:MM #tag3

另一条记录原文。

...

## 〇 数据

- 共记录 N 条
- 涉及标签：tag1(3), tag2(1), tag3(2)
```

## 格式要点

1. **Front-matter**：YAML 格式，包含日期、星期、条目数（`entry_count`）、当日所有去重 Tag 列表
2. **标题**：日期 + 星期
3. **正文**：每条记录为一个三级标题，格式为 `### 时间 #tag1 #tag2`
4. **补录条目**：时间位置显示 `### 补记 #tag1`，排在日志末尾
5. **原文**：标题下紧跟用户原文，保持原样，不添加任何修饰
6. **数据段**：末尾统计当日条目数和各 Tag 出现次数
7. **排序**：按时间正序排列，补记条目排在最后

注意：`entry_count` 是条目总数，`tags` 是去重后的 Tag 列表，两者数量通常不同。

## 原始记录格式（entries）

每日碎片记录存储为 JSONL 文件 `$DIARY_HOME/entries/YYYY-MM-DD.jsonl`，每行一条：

```json
{"time": "09:15", "raw": "和老张开了个需求评审会，把 Q2 的 OKR 对齐了", "tags": ["工作", "会议"]}
{"time": "12:30", "raw": "午饭试了楼下新开的湘菜馆，剁椒鱼头不错", "tags": ["饮食"]}
{"time": "补记", "raw": "昨天忘记录了，下午跑了5公里", "tags": ["运动"]}
```

## 周总结模板

周总结文件为 `$DIARY_HOME/weekly/YYYY-WNN.md`：

```markdown
---
week: YYYY-WNN
range: MM-DD(周一) ~ MM-DD(周日)
total_entries: N
---

# YYYY 第 NN 周总结（MM/DD - MM/DD）

## 〇 概览

本周共记录 N 条，涉及 M 个标签。

## 〇 分类回顾

### #工作
- **周一 09:15** 和老张开了个需求评审会，把 Q2 的 OKR 对齐了
- **周三 10:00** 提交了搜索模块的 PR，等 code review
- ...

### #阅读
- **周二 14:00** 看完了《置身事内》最后两章
- ...

## 〇 标签统计

| 标签 | 条目数 | 占比 |
|------|--------|------|
| 工作 | 12 | 40% |
| 阅读 | 5 | 17% |
| ... | ... | ... |
```

标签统计中，一条记录如果有多个 Tag，每个 Tag 各计一次。占比基于总标记次数计算。
