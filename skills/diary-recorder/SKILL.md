# 马伯庸式日记记录系统

> 体例出自《越缦堂日记》：细大必书，积玉碎金，一日不定几例，一事一条，
> 所读何书，所见何人，所遇何事，皆日常直录。以记事为要，议论次之。

## 配置

本 Skill 使用两个路径，安装后按实际情况修改：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DIARY_HOME` | `~/diary` | 日记数据存放目录 |
| `SKILL_DIR` | 本 Skill 所在目录 | 脚本等资源的引用基准 |

下文中 `$DIARY_HOME` 和 `$SKILL_DIR` 指代上述路径。Agent 执行时将其替换为实际绝对路径。

## 核心原则

1. **用户无感记录**：用户只管说，系统负责一切整理工作
2. **一事一条**：每次用户发来的内容作为一条完整记录，不拆分、不合并
3. **原文保留**：用户原话完整保留（剥离触发词后），整理时仅做结构化，不改写内容
4. **以记事为要**：事实优先，议论附后
5. **明确触发**：只有匹配到触发词才执行记录，避免误判普通对话

## 数据目录

所有日记数据存放在 `$DIARY_HOME` 下，不随工作区变化：

```
$DIARY_HOME/
├── entries/          # 每日原始记录（碎片）
│   └── 2026-03-17.jsonl
├── daily/            # 整理后的每日日记
│   └── 2026-03-17.md
├── weekly/           # 周总结
│   └── 2026-W12.md
├── tags.md           # Tag 列表
└── tag-changelog.md  # Tag 变迁日志
```

## 初始化

首次使用时（`$DIARY_HOME` 目录不存在），自动执行初始化：

1. 创建 `$DIARY_HOME/entries/`、`$DIARY_HOME/daily/`、`$DIARY_HOME/weekly/` 目录
2. 用如下内容初始化 `$DIARY_HOME/tags.md`：

```markdown
# Tag 体系

- 工作
- 会议
- 阅读
- 饮食
- 运动
- 社交
- 健康
- 财务
- 杂项
```

3. 用如下内容初始化 `$DIARY_HOME/tag-changelog.md`：

```markdown
# Tag 变迁日志
```

4. 向用户确认："日记系统已初始化。"

**每次执行任何流程前，先检查 `$DIARY_HOME` 目录是否存在，不存在则先执行初始化。**

## 获取时间

通过脚本获取当前日期和时间：

```bash
python3 $SKILL_DIR/scripts/now.py
```

输出格式：`2026-03-17 14:32 星期二`

每次记录时必须调用此脚本获取准确的本地时间，不要自行推测。

## 触发词与意图判断

**只有明确匹配以下触发词时才执行对应操作，其他对话不做记录。**

| 触发词 | 动作 |
|---|---|
| "记一下…" / "记录…" / "日记：…" | → 流程一「记录」 |
| "整理日记" / "整理今天的" | → 流程二「整理日记」（当天） |
| "整理 MM-DD 的日记" / "整理昨天的日记" | → 流程二「整理日记」（指定日期） |
| "周总结" / "这周总结一下" | → 流程三「周总结」 |
| "补记 MM-DD …" / "补记昨天 …" | → 流程四「补录」 |
| "改一下记录…" / "删掉那条…" | → 流程五「修改/删除」 |
| "看看今天记了什么" | → 读取并展示当天 entries |
| "最近关于 X 的记录" | → 按 Tag 或关键词检索 |

不在上述触发词范围内的用户输入，按正常对话处理，不做日记记录。

## 工作流程

### 流程一：记录（Record）

用户使用触发词发来需要记录的内容时：

1. 调用 `python3 $SKILL_DIR/scripts/now.py` 获取当前日期和时间
2. **剥离触发词**：只保留触发词之后的实际内容作为 `raw`（如"记一下，开了个会" → raw 为"开了个会"）
3. 打开或创建 `$DIARY_HOME/entries/{date}.jsonl`
4. 将记录追加为一行 JSON：

```json
{"time": "HH:MM", "raw": "用户原文（已剥离触发词）", "tags": ["tag1", "tag2"]}
```

5. Tag 匹配规则（详见 [tag-system.md](tag-system.md)）：
   - 从 `$DIARY_HOME/tags.md` 的 Tag 列表中找最接近的，可以打多个
   - 找不到合适的才新增 Tag，并同步更新 `tags.md` 和 `tag-changelog.md`
6. 向用户简短确认："已记录。" 附带时间和分配的 tag，不做多余寒暄

**关键**：记录动作要快，不要反复确认、不要追问细节，用户说什么就记什么。一次发送的内容无论包含多少件事，都作为一条记录保存。

### 流程二：整理日记（Compile）

用户说"整理日记"或"整理 MM-DD 的日记"时：

1. 确定目标日期（默认当天，或用户指定的日期）
2. 读取 `$DIARY_HOME/entries/{date}.jsonl` 的所有记录
3. 按时间排序
4. 按格式模板（见 [diary-format.md](diary-format.md)）生成 `$DIARY_HOME/daily/{date}.md`
5. 每条记录保留原文、时间、Tag，不改写用户原话
6. 在日记末尾附加当日元数据

### 流程三：周总结（Weekly Summary）

用户说"周总结"时：

1. 确定本周范围（周一至周日）
2. 读取该周所有 `$DIARY_HOME/daily/*.md`（如果某天还没整理，先从 entries 读取）
3. 按 Tag 分类汇总，每个 Tag 下列出所有相关条目
4. 按模板生成 `$DIARY_HOME/weekly/{year}-W{week}.md`
5. 每条记录附带日期和时间

### 流程四：补录（Backfill）

用户说"补记"时：

1. 解析目标日期：
   - "补记昨天" → 昨天的日期
   - "补记 03-15" → 2026-03-15（当年）
   - "补记 MM-DD HH:MM" → 指定日期和时间
2. 如果用户没指定时间，`time` 字段记为 `"补记"`
3. **剥离触发词和日期部分**，只保留实际内容作为 `raw`
4. 追加到对应日期的 `$DIARY_HOME/entries/{date}.jsonl`
5. 正常执行 Tag 匹配
6. 如果该日期的 `$DIARY_HOME/daily/{date}.md` 已经生成过，提示用户可以重新整理

### 流程五：修改/删除记录（Edit/Delete）

用户要求修改或删除记录时：

1. 根据用户描述定位目标记录（通过日期、时间、关键词）
2. 如果无法唯一定位，展示候选条目让用户确认
3. **修改**：在 JSONL 中更新该行，保留原始时间
4. **删除**：在 JSONL 中移除该行
5. 如果当天日记已整理，提示用户可以重新整理以同步变更
