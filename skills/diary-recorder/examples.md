# 完整示例

## 示例一：普通记录

### 用户输入

> 记一下，上午和产品开了个会，敲定了 v2.1 的排期

### 系统操作

1. 调用 `python3 $SKILL_DIR/scripts/now.py` → `2026-03-17 10:32 星期二`
2. 剥离触发词"记一下，"，`raw` 为"上午和产品开了个会，敲定了 v2.1 的排期"
3. 读取 `$DIARY_HOME/tags.md`，匹配到 `工作`、`会议`
4. 在 `$DIARY_HOME/entries/2026-03-17.jsonl` 追加：

```json
{"time": "10:32", "raw": "上午和产品开了个会，敲定了 v2.1 的排期", "tags": ["工作", "会议"]}
```

5. 回复：

> 已记录。10:32 #工作 #会议

## 示例二：需要新增 Tag

### 用户输入

> 记录，今天第一次用 Blender 建了个低模角色

### 系统操作

1. 调用 `python3 $SKILL_DIR/scripts/now.py` → `2026-03-17 16:45 星期二`
2. 剥离触发词"记录，"，`raw` 为"今天第一次用 Blender 建了个低模角色"
3. 读取 `$DIARY_HOME/tags.md`，无匹配 → 新增 `建模`
4. 在 `$DIARY_HOME/tags.md` 末尾追加 `- 建模`
5. 在 `$DIARY_HOME/tag-changelog.md` 中：已存在 `## 2026-03-17` 段，追加到该段下：

```markdown
- **新增** `建模`
  - 触发记录: "今天第一次用 Blender 建了个低模角色"
```

6. 在 `$DIARY_HOME/entries/2026-03-17.jsonl` 追加：

```json
{"time": "16:45", "raw": "今天第一次用 Blender 建了个低模角色", "tags": ["建模"]}
```

7. 回复：

> 已记录。16:45 #建模（新增标签）

## 示例三：补录

### 用户输入

> 补记昨天，去医院复查了一下牙齿，没什么问题

### 系统操作

1. 调用 `python3 $SKILL_DIR/scripts/now.py` 获取今天日期，推算昨天为 `2026-03-16`
2. 剥离触发词和日期"补记昨天，"，`raw` 为"去医院复查了一下牙齿，没什么问题"
3. 用户未指定时间，`time` 记为 `"补记"`
4. 匹配 Tag：`健康`、`就医`（就医不存在则新增）
5. 在 `$DIARY_HOME/entries/2026-03-16.jsonl` 追加：

```json
{"time": "补记", "raw": "去医院复查了一下牙齿，没什么问题", "tags": ["健康", "就医"]}
```

6. 检查 `$DIARY_HOME/daily/2026-03-16.md` → 已存在
7. 回复：

> 已补录至 03-16。#健康 #就医（新增标签）
> 该日日记已整理过，如需同步请说"整理 03-16 的日记"。
